# Code Category — Extra Axis: Maintainability

### maintainability (1-5)
Is the generated code readable, well-structured, and easy to change?

- **1 — Unmaintainable**: Spaghetti code, no naming conventions, impossible to follow.
- **2 — Difficult**: Poor naming, no comments, tangled logic; significant effort to modify.
- **3 — Acceptable**: Functional but could be cleaner; some unclear sections.
- **4 — Clean**: Good naming, logical structure, appropriate comments; easy to modify.
- **5 — Exemplary**: Idiomatic, well-documented, follows best practices, easy to extend and test.

When scoring maintainability, consider:
- Are variable and function names descriptive?
- Is the code organized into logical units (functions, classes, modules)?
- Are edge cases handled?
- Is there appropriate documentation (docstrings, comments for non-obvious logic)?
- Does the code follow the conventions of its language?
